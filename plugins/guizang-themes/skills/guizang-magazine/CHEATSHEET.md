# kami · Guizang 杂志风速查

一页速查。填模板、调细节前先看这里。完整规范在 `references/design.md`。

## 范围

- 正式支持：中文 `one-pager`、`long-doc`、`letter`、`slides`
- 兼容保留：`resume`、`portfolio`、英文模板
- 风格目标：**杂志页眉页脚 / 电子墨水反差 / serif 大标题 / sans 正文 / mono 元信息**

## 八条铁律

1. 继续用 Kami 的稳定模板链路，不重做另一套引擎
2. 风格必须一眼像 Guizang，而不是民国文稿或报纸
3. 标题用 serif，正文用 sans，元信息用 mono，三者不能乱
4. 默认只从 5 套预设主题里选，不随手自定义 hex
5. Kicker 和页眉不是同一句话
6. 规则线要细，信息层级靠字体和留白，不靠厚重边框
7. 图片优先 top-safe 裁切，避免把人物头部和主视觉裁掉
8. 页面的节奏要有深浅变化，不能整份文档都长一个样

## 自然语言触发

| 用户说 | 路由 |
|---|---|
| 用歸藏风格做一页项目方案 | `one-pager.html` |
| 把这份白皮书排成杂志风 | `long-doc.html` |
| 推荐信做成电子墨水风 | `letter.html` |
| 做一套 Guizang 风格 slides | `slides.py` |
| 做个杂志风简历 / 作品集 | `resume.html` / `portfolio.html` |

## 主题预设

| 主题 | 用途 |
|---|---|
| 墨水经典 | 默认、通用、最稳 |
| 靛蓝瓷 | AI / 技术 / 数据 / 研究 |
| 森林墨 | 文化 / 非虚构 / 内容型项目 |
| 牛皮纸 | 人文 / 阅读 / 历史 / 叙事 |
| 沙丘 | 设计 / 品牌 / 作品集 |

## 字体分工

| 角色 | 用法 |
|---|---|
| Serif | 大标题、引语、关键数字、章节收束 |
| Sans | 正文、说明、信件主体、图注 |
| Mono | 页眉页脚、标签、日期、页码、元信息 |

## 版式信号

| 想做 | 怎么做 |
|---|---|
| 开场标题 | 大 serif 标题 + kicker + 简短 lead |
| 数据段 | 3-6 个数字卡，标签短、数字大、说明一行 |
| 章节切换 | 大留白 + 规则线 + 页脚节奏变化 |
| 信件 | 头部信息块 + 主题行 + 清晰正文区 |
| 简历 | 大名字 + mono 联系方式 + 横向 metrics |
| 作品集 | 封面强标题 + 项目编号 + 大图优先 |

## 反馈词汇

当用户说“不对劲”，优先确认的是：

- 标题气压
- 纸墨反差
- 栏线密度
- 元信息存在感
- 章节节奏
- 图片裁切

一句话：**内容骨架沿用 Kami，风格信号交给 Guizang 的杂志层级、电子墨水反差和预设主题。**
