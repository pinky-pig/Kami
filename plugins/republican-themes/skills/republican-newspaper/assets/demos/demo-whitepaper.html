封面摘要 / EXECUTIVE BRIEF

从单一主题仓库到可持续扩展的主题 Skill 体系

这份白皮书以虚拟项目 “Kami Demo Lab” 为样本，说明如何把散落的设计风格、模板、脚本和示例产物，整理成可以安装、展示、验证和持续新增的本地 skill 体系。

核心观点是：主题不是一份静态 HTML，也不是一个孤立模板，而是由风格说明、写作策略、资源目录、生成脚本和 demos 共同组成的可执行知识包。它既服务人，也服务 AI 代理。

报纸化排版让信息以“头版、专栏、剪报、号外”的方式被组织。相比普通报告，它更适合展示主题能力、项目案例和方法论摘要，同时保留正式文档的阅读秩序。

体系模型

MODEL · DIRECTORY · GOVERNANCE

一、目录边界

建议将展示层和能力层分离：展示层只负责主题导航、预览和使用入口；能力层则包含每个主题的 SKILL、参考资料、模板、脚本和产物。这样新增主题不会污染前端展示，也不会把交付逻辑散落到根目录。

层级责任虚拟示例
apps/web展示与入口主题目录、demo 链接、安装说明
plugins/kami-themes插件元信息plugin.json、总览 README
skills/*单主题能力包SKILL.md、assets、references、scripts

二、主题最小闭环

  • 风格：用设计说明描述版式、色彩、字体和禁区。
  • 内容：用写作规范限制标题、导语、栏目和注释口吻。
  • 模板：HTML/PPT 模板负责可复制的结构。
  • 产物：demos 目录保留真实可打开的样张。
  • 验证：脚本检查占位符、字体、页数和生成结果。

交付流程

FROM REFERENCE TO DEMO
观察提取参考图中的报头、框线、分栏、纸张和印章。
抽象写成 tokens、design spec、writing guide 和禁用项。
模板制作可替换内容的 HTML、PDF、PPTX 结构。
生成用虚拟数据跑出 resume、whitepaper 和 slide deck。
回归检查无真实个人信息、无占位符、无明显溢出。
摘

一个主题 skill 的价值，不在于复制一张图，而在于让之后的任意内容都能稳定套用同一种审美与结构，并且可以被本地脚本重新生成。

实施路线

ROADMAP · DEMO OPERATIONS

三阶段推进

第 01 周

整理主题源材料

收集参考图、提炼关键词、拆出纸张、字体、版式和栏目结构，形成最初的 design notes。

第 02 周

落地模板与示例

以真实排版结构生成 one-pager、resume、whitepaper、pptx 等 demo，使用虚拟数据避免隐私泄露。

第 03 周

接入展示与验证

把 demos 放进 web 入口，补充脚本检查与生成命令，让新增主题可以按同一流程复制。

验收清单

  • 打开 HTML 时，首页能直接看到该主题的核心视觉信号。
  • 导出的 PDF 页面尺寸正确，背景、框线和字体完整。
  • PPTX 文字可编辑，不依赖整页截图。
  • demo 内容使用虚拟姓名、邮箱、组织和指标。
  • 后续新增主题时，只需要复制 skill 目录并替换资源。

结论剪报

CONCLUSION CLIPPING
结论一

主题即能力包

主题应包含执行说明、模板、参考、脚本和示例，而不是只保留一个静态页面。

结论二

demo 即验收

resume、whitepaper、pptx 等 demo 覆盖不同内容密度，是判断主题是否可复用的直接证据。

结论三

隐私先行

公开样张默认使用虚拟数据，真实姓名、联系方式和公司信息只在正式交付版本中替换。

结论四

持续扩展

当目录边界清楚后，新增 n 个主题 skill 会变成重复可靠的生产动作。
